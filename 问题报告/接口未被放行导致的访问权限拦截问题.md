# 民宿预订系统 403 错误问题分析与解决报告

---

## 一、问题描述

在民宿预订系统开发过程中，前端访问推荐民宿接口时出现 403 错误。

### 报错信息

前端控制台显示：

```
GET http://localhost:3000/api/homestay/recommend 403 (Forbidden)
```

浏览器直接访问后端接口：

```
http://localhost:8080/api/homestay/recommend
```

返回：

```
HTTP ERROR 403
您未获授权
```

---

## 二、问题现象分析

### 1️⃣ 请求路径确认

接口路径为：

```
/api/homestay/recommend
```

Controller 配置如下：

```java
@RestController
@RequestMapping("/api/homestay")
public class HomestayController
```

推荐接口：

```java
@GetMapping("/recommend")
public Result getRecommendedHomestays()
```

接口路径匹配正确。

---

### 2️⃣ JWT 过滤器检查

JwtAuthenticationFilter 逻辑：

* 若存在合法 Token → 设置认证信息
* 若无 Token → 直接放行

代码无强制拦截逻辑，不会导致 403。

---

### 3️⃣ SecurityConfig 安全配置检查

原始安全配置：

```java
.antMatchers("/api/auth/**").permitAll()
.antMatchers("/api/geocoding/**").permitAll()
.antMatchers("/api/city/**").permitAll()
.antMatchers("/api/cities/**").permitAll()
.antMatchers("/api/tag/**").permitAll()
.antMatchers("/api/tags/**").permitAll()
.antMatchers("/uploads/**").permitAll()
.antMatchers("/static/**").permitAll()

.anyRequest().authenticated();
```

问题点：

```
/api/homestay/** 未被放行
```

因此：

```
/api/homestay/recommend
```

被 `.anyRequest().authenticated()` 拦截。

未登录访问 → 返回 403 Forbidden。

---

## 三、问题根本原因

由于 Spring Security 默认规则：

```java
.anyRequest().authenticated();
```

要求所有未明确放行的接口必须登录访问。

推荐接口未加入 `permitAll()` 列表，导致未登录用户访问时被安全框架拦截。

---

## 四、解决方案

在 `SecurityConfig.java` 中添加放行规则：

```java
.antMatchers("/api/homestay/**").permitAll()
```

修改后完整配置示例：

```java
@Override
protected void configure(HttpSecurity http) throws Exception {
    http
        .csrf().disable()
        .cors()
        .and()
        .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class)
        .authorizeRequests()

        .antMatchers("/api/auth/**").permitAll()
        .antMatchers("/api/geocoding/**").permitAll()
        .antMatchers("/api/city/**").permitAll()
        .antMatchers("/api/cities/**").permitAll()
        .antMatchers("/api/tag/**").permitAll()
        .antMatchers("/api/tags/**").permitAll()

        // 放行民宿相关接口
        .antMatchers("/api/homestay/**").permitAll()

        .antMatchers("/uploads/**").permitAll()
        .antMatchers("/static/**").permitAll()

        .anyRequest().authenticated();
}
```

修改后：

1. 重启后端服务
2. 清除浏览器缓存
3. 重新访问接口

问题成功解决。

## 六、经验总结

### 1️⃣ Spring Security 默认行为

`.anyRequest().authenticated()` 会拦截所有未放行接口。

必须明确配置允许匿名访问的接口。

---

### 2️⃣ 接口权限设计建议

建议采用：

* 游客可访问：列表、详情、搜索、推荐
* 普通用户：下单、收藏
* 房东：发布、修改、删除
* 管理员：后台管理

权限控制建议：

* 公共接口使用 `permitAll()`
* 业务权限使用 `@PreAuthorize`

---

### 3️⃣ 排查思路总结

本次排查过程：

1. 确认前端代理
2. 确认接口路径
3. 排除 JWT 过滤器问题
4. 定位 SecurityConfig 权限匹配问题
5. 修改并重启验证

属于典型的 Spring Security 权限配置遗漏问题。

---

## 七、最终结论

本次 403 错误是由于：

> Spring Security 未放行推荐接口，导致未登录访问被拦截。

通过在 SecurityConfig 中添加接口放行规则，问题成功解决。
