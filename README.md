
这是一个基于现代Web技术栈开发的民宿预订系统，分为前端和后端两部分，支持用户、房东和管理员三种角色。

技术栈
前端
● 框架: Vue 3 (Composition API) + <script setup>语法
● 构建工具: Vite
● UI组件库: Element Plus
● 状态管理: Pinia
● 路由: Vue Router
● HTTP客户端: Axios
● 地图服务: 高德地图 API
后端
● 框架: Spring Boot 2.7.17
● ORM: MyBatis Plus 3.5.3.1
● 数据库: MySQL 8.0
● 安全认证: JWT (JSON Web Token)

核心功能
1. 用户认证与角色系统
● 支持三种角色：游客（TOURIST）、房东（LANDLORD）、管理员（ADMIN）
● 用户注册、登录（支持用户名/手机号）
● Token自动刷新机制
● 路由权限守卫
2. 民宿管理
● 游客功能：
  ○ 浏览民宿列表（分页展示）
  ○ 多条件搜索（关键词、价格、标签、热门城市）
  ○ 查看民宿详情（图片轮播、设施、位置地图、房东信息等）
  ○ 预订民宿（选择入住/退房日期，自动计算价格）
● 房东功能：
  ○ 发布民宿（完整表单，支持地图定位和图片上传）
  ○ 管理自己的民宿（查看、编辑、删除）
  ○ 查看和管理订单
● 管理员功能：
  ○ 审核民宿（发布和修改都需要审核）
  ○ 审核特色体验项目
  ○ 数据统计（待审核民宿、待审核体验等）
3. 特色体验（乡村民俗体验）
● 房东可以创建与民宿相关的特色体验项目（如农事、手工等）
● 用户可以预约体验项目
● 体验项目也需要管理员审核后才能展示
4. 订单系统
● 用户预订后生成订单
● 订单状态管理：待支付 → 已支付 → 已完成 → 已取消
● 支付功能（模拟支付）
● 订单取消
● 订单完成后的评价功能
5. 通知与消息
● 订单状态变更通知
● 审核结果通知

主要特色
1. 完整的审核流程：
  ○ 房东发布/修改民宿 → 管理员审核 → 审核通过后上架
  ○ 房东创建特色体验 → 管理员审核 → 审核通过后展示
2. 地图集成：
  ○ 使用高德地图API
  ○ 发布民宿时支持地图定位
  ○ 民宿详情页显示位置地图
3. 响应式设计：
  ○ 支持移动端和桌面端
  ○ 美观的UI设计（参考爱彼迎风格）
4. 角色分离：
  ○ 不同角色有不同的功能权限
  ○ 清晰的用户、房东、管理员界面
这是一个功能完整、架构清晰的民宿预订系统，涵盖了从用户浏览预订到房东管理、管理员审核的完整业务流程。
