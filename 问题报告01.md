# 问题报告01

## 1. SpringBoot项目依赖爆红问题
**问题描述**：在IDEA中开发SpringBoot项目时，所有依赖包都显示红色错误。
**原因分析**：可能是Maven依赖未正确下载或配置问题。
**解决方法**：
1. 确保IDEA已正确配置Maven路径
2. 右键项目 -> Maven -> Reimport
3. 检查pom.xml文件中的依赖配置是否正确

## 2. Maven命令无法识别问题
**问题描述**：在命令行执行`mvn -version`时，系统提示"无法将'mvn'项识别为 cmdlet、函数、脚本文件或可运行程序的名称"。
**原因分析**：Maven未配置到系统环境变量中，或者环境变量配置错误。
**解决方法**：
1. 检查Maven安装目录
2. 将Maven的bin目录添加到系统环境变量PATH中
3. 重启命令行窗口后重试

## 3. Java不支持发行版本5问题
**问题描述**：编译Java项目时出现"java: 错误: 不支持发行版本 5"错误。
**原因分析**：Maven默认使用Java 5编译器，而项目需要更高版本的Java。
**解决方法**：
在pom.xml文件中添加Maven编译器配置：
```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.8.1</version>
            <configuration>
                <source>1.8</source>
                <target>1.8</target>
            </configuration>
        </plugin>
    </plugins>
</build>
```

## 4. Lombok程序包不存在问题
**问题描述**：编译时出现"java: 程序包lombok不存在"错误。
**原因分析**：缺少Lombok依赖或IDE未安装Lombok插件。
**解决方法**：
1. 在pom.xml中添加Lombok依赖：
   ```xml
   <dependency>
       <groupId>org.projectlombok</groupId>
       <artifactId>lombok</artifactId>
       <version>1.18.24</version>
       <scope>provided</scope>
   </dependency>
   ```
2. 在IDEA中安装Lombok插件并重启IDEA

## 5. Leaflet地图加载超时问题
**问题描述**：前端使用Leaflet加载OpenStreetMap瓦片时，出现"net::ERR_CONNECTION_TIMED_OUT"错误。
**原因分析**：OpenStreetMap的默认瓦片服务器在国内访问不稳定。
**解决方法**：
使用国内可访问的地图瓦片服务器，如OpenStreetMap France HOT：
```javascript
L.tileLayer('https://{s}.tile.openstreetmap.fr/hot/{z}/{x}/{y}.png', {
    maxZoom: 18,
    minZoom: 5,
    attribution: '© OpenStreetMap contributors, Tiles style by Humanitarian OpenStreetMap Team hosted by OpenStreetMap France'
}).addTo(map)
```

## 6. TypeError: Cannot read properties of undefined (reading '_leaflet_pos')问题
**问题描述**：在HomestayMap组件中，当组件销毁后，窗口resize事件仍在尝试访问已销毁的地图实例，导致出现该错误。
**原因分析**：
1. 地图组件销毁时，未正确移除窗口resize事件监听器
2. 事件监听器使用了匿名函数，无法正确移除
3. 地图实例销毁后，事件监听器仍持有对该实例的引用
**解决方法**：
1. 创建命名的事件处理函数
2. 在onMounted中添加事件监听器
3. 在onUnmounted中正确移除事件监听器
4. 添加try-catch块防止访问已销毁的地图实例
5. 在销毁地图实例后将其引用设为null
```javascript
// 创建命名的事件处理函数
const handleResize = () => {
  if (map.value) {
    try {
      map.value.invalidateSize()
    } catch (error) {
      console.warn('地图已被销毁，忽略invalidateSize调用')
    }
  }
}

onMounted(() => {
  initMap()
  window.addEventListener('resize', handleResize)
})

onUnmounted(() => {
  // 先移除事件监听
  window.removeEventListener('resize', handleResize)
  
  // 清理地图实例
  if (map.value) {
    map.value.remove()
    map.value = null // 确保map.value不再引用已销毁的实例
  }
})
```

## 7. 房东修改房源信息后没有更新问题
**问题描述**：房东在编辑房源信息后，修改内容没有正确保存到房源列表中。
**原因分析**：
1. 保存房源时，直接使用表单数据替换了原始房源对象
2. 表单数据中缺少原始房源的部分字段（如status、createdTime等）
3. 导致这些字段在更新时丢失
**解决方法**：
在保存房源时，使用对象展开运算符合并原始房源字段和表单字段，确保保留所有未修改的字段：
```javascript
// 保存房源
const saveHomestay = (homestayData) => {
  if (editingHomestay.value) {
    // 更新现有房源
    const index = homestays.value.findIndex(item => item.id === editingHomestay.value.id)
    if (index > -1) {
      // 保留原始房源的所有字段，只更新表单中修改的字段
      homestays.value[index] = {
        ...homestays.value[index], // 保留原始字段
        ...homestayData,           // 更新表单字段
        id: editingHomestay.value.id // 确保id不变
      }
      ElMessage.success('房源更新成功！')
    }
  } else {
    // 添加新房源逻辑...
  }
  handleDialogClose()
}
```

## 8. TypeError: Cannot read properties of undefined (reading 'length')问题
**问题描述**：在HomestayMap组件中，筛选民宿时出现"TypeError: Cannot read properties of undefined (reading 'length')"错误。
**原因分析**：
1. 在filterHomestays函数中，直接访问了filterForm.tags.length
2. 但filterForm是一个ref对象，需要通过.value访问其值
**解决方法**：
在访问ref对象的属性时，正确使用.value：
```javascript
// 筛选民宿
const filterHomestays = () => {
  // 正确访问ref对象的value属性
  ElMessage.info(`已筛选 ${filterForm.value.tags.length} 个标签`)
  // 实际项目中，这里应该调用后端API获取符合条件的民宿
}

// 重置筛选
const resetFilter = () => {
  // 正确访问ref对象的value属性
  filterForm.value.tags = []
  ElMessage.info('筛选条件已重置')
}
```
