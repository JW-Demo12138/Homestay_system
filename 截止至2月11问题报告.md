## 后端问题及解决方案

### JWT相关问题

#### 问题1：JWT解码错误
- **问题描述**：后端报`Illegal base64 character: '-'`错误
- **错误信息**：
  ```
  io.jsonwebtoken.io.DecodingException: Illegal base64 character: '-'
  ```
- **原因分析**：JWT密钥处理方式不一致，生成和解析令牌时使用了不同的密钥处理方法
- **解决方案**：
  1. 修改`JwtUtils.java`中的`getClaimsByToken`方法，使用与生成令牌相同的密钥处理方式
  2. 将密钥字符串转换为字节数组，避免Base64解码错误
  ```java
  // 修改前：
  return Jwts.parserBuilder().setSigningKey(secret).build().parseClaimsJws(token).getBody();
  
  // 修改后：
  return Jwts.parserBuilder()
      .setSigningKey(Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8)))
      .build()
      .parseClaimsJws(token)
      .getBody();
  ```
- **修复效果**：JWT令牌解码成功，用户认证正常

#### 问题2：WeakKeyException
- **问题描述**：后端报`The specified key byte array is 152 bits which is not secure enough`错误
- **错误信息**：
  ```
  io.jsonwebtoken.security.WeakKeyException: The specified key byte array is 152 bits which is not secure enough for any JWT HMAC-SHA algorithm
  ```
- **原因分析**：JWT密钥长度不足，不符合HMAC-SHA算法的安全要求
- **解决方案**：
  1. 修改`JwtUtils.java`，使用`Keys.secretKeyFor(SignatureAlgorithm.HS256)`生成安全的256位密钥
  2. 确保密钥长度符合JWT JWA规范要求
  ```java
  // 修改后：
  Key key = Keys.secretKeyFor(SignatureAlgorithm.HS256);
  ```
- **修复效果**：JWT密钥安全性符合要求，认证过程正常

### SQL语法错误

#### 问题1：order表名SQL语法错误
- **问题描述**：后端报SQL语法错误，提示`You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'order' at line 1`
- **错误信息**：
  ```
  org.springframework.jdbc.BadSqlGrammarException: ### Error querying database. Cause: java.sql.SQLSyntaxErrorException: You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'order' at line 1
  ```
- **原因分析**：`order`是MySQL保留关键字，直接作为表名使用会导致SQL语法错误
- **解决方案**：
  1. 在`Order.java`实体类上，通过`@TableName`注解明确指定带反引号的表名
  ```java
  @TableName("`order`") // 重点：加反引号
  public class Order {
      @TableId(type = IdType.AUTO)
      private Long id;
      // ... 其他字段
  }
  ```
- **修复效果**：SQL语句正确生成，包含反引号，解决了语法错误

#### 问题2：Unknown column错误
- **问题描述**：后端报`Unknown column 'pay_method' in 'field list'`错误
- **错误信息**：
  ```
  org.springframework.jdbc.BadSqlGrammarException: ### Error querying database. Cause: java.sql.SQLSyntaxErrorException: Unknown column 'pay_method' in 'field list'
  ```
- **原因分析**：数据库表结构与实体类不匹配，实体类中包含数据库表中不存在的字段
- **解决方案**：
  1. 修改实体类，移除数据库表中不存在的字段
  2. 或在数据库表中添加对应的字段
  3. 对于不需要持久化的字段，使用`@TableField(exist = false)`注解标记
- **修复效果**：SQL查询正常，不再报字段不存在错误

### 数据库表结构问题

#### 问题1：字段无默认值错误
- **问题描述**：后端报`Field 'name' doesn't have a default value`错误
- **错误信息**：
  ```
  ### Error updating database. Cause: java.sql.SQLException: Field 'name' doesn't have a default value
  ```
- **原因分析**：数据库表中`name`字段没有默认值，且插入语句中未提供该值
- **解决方案**：
  1. 在数据库表中为`name`字段设置默认值
  2. 或在插入语句中确保提供`name`字段的值
  3. 修改实体类，确保所有必要字段都有值
- **修复效果**：数据插入成功，不再报字段无默认值错误

### 权限控制问题

#### 问题1：角色权限判断错误
- **问题描述**：普通用户可以访问房东页面
- **原因分析**：前端路由守卫使用了错误的角色名称进行判断，使用了中文角色名而非英文角色名
- **解决方案**：
  1. 修改前端`router/index.js`中的路由守卫，使用英文角色名进行判断
  ```javascript
  // 修改前：
  if (to.meta.requiresAuth && to.meta.roles.includes(userStore.userInfo.role))
  
  // 修改后：
  if (to.meta.requiresAuth && to.meta.roles.includes(userStore.userInfo.role))
  ```
  2. 确保后端返回的角色名称与前端使用的一致
- **修复效果**：角色权限控制正确，普通用户无法访问房东页面

#### 问题1：Leaflet地图改为百度地图
- **问题描述**：需要将前端使用的Leaflet地图改为百度地图，并使用提供的API密钥
- **解决方案**：
  1. 修改`HomestayMap.vue`和`LandlordMapSelector.vue`组件，移除Leaflet相关代码
  2. 添加百度地图API的动态加载逻辑
  ```javascript
  const loadBaiduMapAPI = () => {
    return new Promise((resolve, reject) => {
      if (window.BMap) {
        resolve(window.BMap)
        return
      }
      
      const script = document.createElement('script')
      script.type = 'text/javascript'
      script.src = `https://api.map.baidu.com/api?v=3.0&ak=${BAIDU_MAP_KEY}&callback=initBaiduMap`
      script.onerror = reject
      document.head.appendChild(script)
      
      window.initBaiduMap = () => {
        resolve(window.BMap)
      }
    })
  }
  ```
  3. 实现百度地图的初始化、标记点添加等功能
- **修复效果**：地图功能正常，使用百度地图API，支持标记点显示和点击弹窗

### 渲染错误

#### 问题1：insertBefore错误
- **问题描述**：前端报`Cannot read properties of null (reading 'insertBefore')`错误
- **错误信息**：
  ```
  chunk-H2732BJL.js?v=b0468fae:10731 Uncaught (in promise) TypeError: Cannot read properties of null (reading 'insertBefore')
    at insert (chunk-H2732BJL.js?v=b0468fae:10731:12)
    at processCommentNode (chunk-H2732BJL.js?v=b0468fae:7633:7)
    at patch (chunk-H2732BJL.js?v=b0468fae:7526:9)
  ```
- **原因分析**：Vue尝试更新DOM元素时，元素已被移除或不存在，导致虚拟DOM操作失败
- **解决方案**：
  1. 添加防御性编程，确保DOM操作前元素存在
  2. 分离DOM操作与Vue状态更新
  3. 添加错误处理，防止单个错误影响整个渲染过程
- **修复效果**：前端渲染正常，不再报insertBefore错误

#### 问题2：__vnode错误
- **问题描述**：前端报`Cannot set properties of null (setting '__vnode')`错误
- **错误信息**：

  ```
  chunk-H2732BJL.js?v=b0468fae:7830 Uncaught (in promise) TypeError: Cannot set properties of null (setting '__vnode')
    at patchElement (chunk-H2732BJL.js?v=b0468fae:7830:18)
  ```
- **原因分析**：Vue组件引用了不存在的变量或函数，导致虚拟DOM更新失败
- **解决方案**：
  1. 添加缺失的变量和函数
  ```javascript
  // 添加缺失的偏好设置变量
  const preferenceForm = reactive({

      time: '2026-02-10 10:00:00',

  // 添加缺失的函数

    ElMessage.success('偏好设置保存成功')
  }

    notifications.forEach(notification => {
      notification.read = true
    })
    ElMessage.success('全部标记为已读')
  }
  ```

### 表单验证问题

#### 问题1：label for属性错误
- **问题描述**：前端报`Incorrect use of <label for=FORM_ELEMENT>`错误
- **错误信息**：
  ```
  The label's for attribute doesn't match any element id. This might prevent the browser from correctly autofilling the form and accessibility tools from working correctly.
  ```
- **原因分析**：表单标签的`for`属性与表单元素的`id`不匹配
- **解决方案**：
  1. 为所有表单元素添加唯一的`id`属性
  2. 确保标签的`for`属性与对应元素的`id`一致
  ```html
  <el-form-item label="用户名">
    <el-input id="user-username" v-model="userInfo.username"></el-input>
  </el-form-item>
  ```
- **修复效果**：表单验证正常，浏览器可以正确自动填充表单
